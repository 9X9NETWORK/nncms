在使用 JsFormat 時，有幾個比較有問題的注意事項如下：
1. JsFormat 會將多行物件屬性中的上下垂直對齊空白全部移除，例如 config.js 裡的 API_PACK 設定。
2. JsFormat 會把多行變數指派中的上下垂直對齊空白全部移除。
3. JsFormat 會移除單行程式後方，與其後註解之間的空白，與註解之間只會留下一個空白。
4. JsFormat 會讓陣列物件屬性內縮 2 層，但 JSLint 預期所有階層都只能再往內縮 1 層，JSLint 可透過 /*jslint white: true */ 關閉此項檢查。
5. JsFormat 會在 function 註解下方與 function 區塊開頭上方之間，強制多插入一行空行。
6. JsFormat 強制 function 區塊的上方，至少一定要保留一行空行。
7. JsFormat 會把連續鏈性使用函數時，代表階層關係的斷行及縮排，全部移除縮排只保留斷行 (例如 func.curation.js)。

9. JsFormat 會把 if 判斷式本身多個條件間的斷行，全部只會內縮 1 層 (前提是以二元運算子為結束，建議 follow JsFormat 全部合併為一行)。
   JSLint 預期在這種狀況下應該內縮 2 層，以便和判斷式內的程式內縮 1 層作區隔。
10. JsFormat 會把多行字串串接之間的分組斷行，全部合併為一行。
